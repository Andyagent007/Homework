  1. Prove that there are infinitely many prime numbers.

    answer

                        Assume there are only a finite quantity of primes
                    then let q = the biggest prime number
                    2, 3, 5, 7, .... q these are all of the prime number in the universe
                    p = 2*3*5*7...*q
                    let N = p+1
                    is N divisible by 2? no
                    is N divisible by 3? no
                    is N divisible by 5? no
                    is N divisible by 7? no
                    .
                    .
                    .
                    is N divisible by q? no

                    N is not divisible by any prime smaller than it, so it must be prime.
                    But this is a contradiction, because q was supposed to be the biggest prime.
                    And N is biggest than q, and also it is prime. 
                    Therefore our assumption is false, so there are in fact infinitely many primes.

  11. What is "nullish coalescing", the new operator ?? introduced into JavaScript in the year 2020? Do some research/reading, then explain these:

    answer

    The nullish coalescing operator (??) it's similar to the OR (||) operator, just that the nullish coalescing operator only recognize as empty the values null or undefined. It means that it only will return the rigth value if the left value is null or undefined, diferent as the OR operator that will return the rigth value if the left is a falsy value (null, undefined, false, 0, NaN and ""

    1. x = null; y = x ?? "hello"; = It would return the "hello", because x is null then it will return the rigth value.
    2. x = undefined; y = x ?? "hello"; = It will return "hello", because x is undefined ,so it return the rigth value.
    3. x = 3.14; y = x ?? "hello"; = In this one x has a value of 3.14, then it will return the left side because x is not empty.
    4. x = false; y = x ?? "hello"; = In this case we have that x is false, and the rules say that the nullish coalescing operator only doesn't accept null and undefined, so y must be false.
    5. x = NaN; y = x ?? "hello"; = In this case y will be NaN, because NaN is a falsy value.
    6. x = ""; y = x ?? "hello"; = y is equal to "", because although x is with empty quotes, it is not either null or undefined
    7. x = []; y = x ?? "hello" = This time the nullish coalescing operator will return [], the same as all ones the [] are not null or undefined
    8. x = 0; y = x ?? "hello"; = If x is 0, y would be 0 also because x has a value.
    9. x = {}; y = x ?? "hello"; = The same in this one, x is {}, then the nullish coalescing operator would return the rigth value.
